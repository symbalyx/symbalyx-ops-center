Problème identifié

n8n v2.22.6 bloque les appels réseau dans les nœuds Code — fetch(), require(), $http sont interdits par le sandbox de sécurité. Les 10 workflows FLA utilisaient tous une fonction safeFetch() qui échouait silencieusement, rendant tous les webhooks inopérants (réponse 200 vide).

Solution appliquée

Remplacer chaque appel réseau (fetch → Ollama / Qdrant) par des nœuds HTTP Request natifs n8n. Architecture en 3 étapes :

Code (parse & validation pure JS)
  ↓
NoOp ──→ HTTP Request (Ollama / Qdrant)
  ↓                ↓
Merge ←───────────┘ (préserve les métadonnées)
  ↓
Code (construction du payload)
  ↓
HTTP Request (stockage Qdrant)
  ↓
Code (réponse formatée)
